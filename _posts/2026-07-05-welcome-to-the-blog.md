This is the first post. It exists to show you the format you'll write in, and
to confirm the feed is working. Delete it whenever you're ready to publish for real.

## What this space is for

Short, practical notes on the things I work on every day: AI adoption and
governance in higher education, student-centered systems, and what it actually
takes to get people to trust and use new tools. No hype — just what's holding up
in practice.

## How to add a new post

1. Create a file in the `_posts/` folder.
2. Name it `YYYY-MM-DD-a-short-slug.md` — for example, `2026-08-01-notes-on-ai-governance.md`. The date in the filename is what orders the blog.
3. Start the file with the front-matter block (the part between the two `---` lines), then write the body in Markdown.
4. Commit and push. GitHub Pages rebuilds the site and the feed automatically — usually within a minute.

## The front matter

Every post begins with a small block of settings between two `---` lines, exactly
like the top of this file:

- **title** — shown as the headline and in the feed.
- **date** — the publish date (and optional time).
- **description** — one sentence used for the blog-list preview, the page's meta description, and the feed summary. Optional, but recommended.

You don't need to set `layout` or the URL — the site is configured to give every
post the shared design and a clean `/blog/your-slug/` address on its own.

## Formatting you can use

Markdown handles the rest. You can write **bold** and *italic* text, add
[links](https://www.daricecorey.com/), and drop in lists:

- A first point
- A second point
- A third point

> Blockquotes look like this — handy for pulling out a key line.

That's everything. Change the title, write your thoughts below the front matter,
and you have a post.
