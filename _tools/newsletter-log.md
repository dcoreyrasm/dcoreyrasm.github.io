# Darice on AI, issue log

Purpose: track issues so themes and per-tool features do not repeat. Read this log before
choosing a theme, then add a line after each issue.

This log lives in the site repo (not in Dropbox and not in the skill) on purpose, so it is
version-controlled and readable from any context, including a scheduled cloud agent that
only has the GitHub repo. It is under `_tools/`, which Jekyll ignores, so it is never served.

## How to use this file

Before choosing a theme, read "Issues so far" and pick a theme and four features not already
listed. Vary the per-tool feature even when a theme area returns, so no single feature repeats.
After an issue is final, append one line to "Issues so far" (and add the theme to "Do not
reuse"), then commit it with the post. A single line per issue — no more dated-copy files.
